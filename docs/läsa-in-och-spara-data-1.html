Kapitel 6 Läsa in och spara data

Det finns många typer av dataformat och det går inte att täcka alla. Här går vi igenom de vanligaste typerna, som samtidigt ger principer för hur data. Det är bra att vara medveten om att datainläsning kan i en del fall ta mycket tid om data från början inte organiserat på ett bra sätt.

Det enklaste sättet att importera data till R är från en textfil. Genom att spara data i textformat blir data materialet dessutom program- och plattformsoberoende. Det är viktigt att organisera data på ett bra sätt.

  • Spara data som en text-fil
  • Många program (t ex Excel) gör det möjligt att spara data som ren text.
  • Organisera data med kolumner som korresponderar till olika variabler innan du sparar data till en textfil
  • Använd semi-kolon för att avgränsa (delimit) kolumner. Komma används som decimaltecken på svenska och avstånd eller TAB är svåra att se i en textfil eftersom de påminner om blanksteg.
  • Försök att inte ändra något i ursprungsfilen. Det är bättre att, om möjligt, importera till R och göra ändringarna där.

6.0.1 CSV-filer

CSV står för “comma separated values”, vilket även referar till semi-kolon eftersom många länder använder komma som decimaltecken. De två funktionerna som används för att läsa in CSV-filer är read.csv() och read.csv2() och mer generellt text-filer read.table(). De bägge read.csv() och read.csv2() är nämligen scpecial av read.table(), men med förspecificerade argument. Funktionerna har dessa utseende:

read.csv(file, header = TRUE, sep = ",", quote = "\"", dec = ".")
read.csv2(file, header = TRUE, sep = ";", quote = "\"", dec = ",")
read.table(file, header = TRUE, sep=" ", quote = "\"", dec = ".")
  • file anger filnamnet
  • header om variablerna har variabelnamn på första raden
  • sep anger typ av separarering mellan kolumner.
  • quote anger citationstecken för text-variabler
  • dec anger typ av decimal-tecken

6.0.2 Excel-filer

I praktiken kommer ni stöta på import av Excel-dokument. För ett antal tips när data matas in i Excel.

  • Första raden ska bestå av variabelnamn
  • Första kolumnen bör bestå av observationsid
  • Undvik namn och värden med mellanslag. Risken är att det tolkas som två olika kolumner.
  • Korta variabelnamn är bättre än långa.
  • Undvik tecken som ?, $,%, ^, &, *, (, ),-,#, ?,,,<,>, /, |, , [ ,] ,{, and };
  • Radera kommentarer som du har i Excelfilen så att dessa inte tolkas som extra data.

Prova nu läsa in Excel-filerna animal3.xlsx och animal4.xlsx.

Installera ett paket som heter

# Installerar paket. Behövs bara göras en gång
install.packages("readxl")
# Läser in paket i en R-session. Måste göras varje
# gång en session startar om en funktion i paketet skal # användas.
library("readxl)
df5 <- read_excel("animal3.xlsx")

Om du nu tittar på datamaterialet så ska det inte vara något problem.

df6 <- read_excel("animal4.xlsx")

Det är nu något som inte stämmer. Om öppnar filen med Excel så kan du se att Excel-dokumentet består av flera kalkylblad. Data finns i blad 2. Dessutom börjar inte data från toppen, utan i blad två finns lite annan text. Genom att vara mer specifik i inläsningen av data väljer vi varifrån Excel-filen data hämtas.

df7 <- read_excel("animal4.xlsx", sheet = "Blad2", range = "A3:D68")

6.0.3 Övriga format

Det finns ett antal specialfall som vi inte tar upp. Främst gäller det:

  • import av andra format (t ex SAS-format, SPSS-format, Minitab). Detta kan lösas med exempelvis paket som foreign.
  • import av stora dataset, där till exempel paketet data.table kan vara till hjälp.
